Welcome to the mustelinet tenant documentation. These pages explain how to use the Skyline dashboard, create tenant resources, log in to VMs through Skyline console or mustelinet SSH, publish applications through generated DNS names, and automate repeatable environments with Terraform.

The docs are written for cloud tenants. They avoid platform operator procedures and focus on the actions you can take in your own OpenStack project.

## Common tenant tasks

- Learn where Skyline shows projects, instances, images, flavors, networks, routers, floating IPs, volumes, Kubernetes clusters, and application credentials.
- Create Linux instances with networks, security group rules, and optional SSH keypairs.
- Use Skyline console login with a VM password when the VM should not expose SSH.
- Use generated `apps.mustelinet.com` DNS names as the stable public names for VMs and load balancers.
- Reach tenant web apps through the public edge on HTTP or HTTPS.
- Connect to VMs through mustelinet SSH when the VM and project policy allow it.
- Create application credentials for Terraform, CI, and API automation.
- Use Terraform for repeatable stacks, managed Kubernetes clusters, and team-managed infrastructure.
